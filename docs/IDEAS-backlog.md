# Бэклог идей улучшений

Сюда складываем согласованные идеи, чтобы не потерялись и можно было «если что добавить».
Источник — обсуждение 2026-06-16. Статусы: `⬜ не начато` · `🟡 в работе` · `✅ сделано` · `⏸ отложено`.

## Принцип-фильтр (важно)

Новые данные о стопе/обуви берём **только** там, где есть честный источник
(известная фактура **или** ввод пользователя) **И** где они реально меняют вердикт
обоснованно. Данные «для красоты», которые ничего честно не уточняют, — это лишняя
поверхность для галлюцинаций. Через этот фильтр и отобраны пункты ниже.

## Приоритет 1 — быстрый выигрыш

### ⬜ Позитивные инструкции замера (UX)
Подписи у полей замера сейчас говорят «как НЕ надо» («это НЕ длина стельки и НЕ размер»).
Переписать на «как НАДО»:
> Как измерить: встань на лист бумаги, обведи стопу карандашом вертикально, измерь
> линейкой от пятки до самого длинного пальца (в см). Мерь вечером и обе стопы — бери большую.
- **Честность/ценность:** меньше ошибок ввода → честнее результат. Усилия крошечные.
- **Где:** `src/components/FootInput.tsx` (подписи), готово к применению хоть сейчас.

## Приоритет 2 — вместе с шагом 4 (фидбэк словами)

### ⬜ Материал → растяжимость / разнашивание
Курируемая таблица «материал → поведение» (кожа: тянется, разнашивается; трикотаж/knit:
гибкий, облегает; синтетика/TPU: жёсткий, не даёт; замша: немного тянется). Питает
смягчение вердикта по `stretch` и честную ремарку «тесно по ширине, но кожа разносится».
- **Честность/ценность:** снижает галлюцинации — `stretch` берём из правила, а не из догадки LLM.
  Высокая ценность. Ремарку про разнашивание привязывать СТРОГО к материалу (knit/синтетика
  не разнашиваются — иначе враньё).
- **Где:** шаг 4 (вердикт), нормализация строк `upperMaterial`. Усилия низкие-средние.

## Приоритет 3 — отдельная фича, высокая ценность

### ⬜ Сравнение с обувью, которую уже удобно носишь
Пользователь указывает модель/размер обуви, которую комфортно носит → якорь на **реальные
данные** для калибровки вердикта («ты удобно носишь X EU 42, эта модель малит → бери 43»).
- **Честность/ценность:** высшая честность — реальный якорь против закрытых колодок.
  Прямо заявлено сильной стороной продукта. Самый сильный рычаг из всех идей.
- **Где:** отдельная спека (хранение «эталонной» обуви + сопоставление). Усилия средние.

## Приоритет 4 — после шага 4

### ⬜ Тип стопы по пальцам (египетская / греческая / римская)
Какой палец длиннее. Закрывает дыру геометрии длина×ширина: где давит носок (греческая
стопа + узкий/низкий toe-box = давление на 2-й палец).
- **Честность/ценность:** пользователь сам определяет тип по картинке; маппинг «тип + toeBox →
  последствие» защитимый. Средняя ценность, средние усилия (вариация силуэта + логика носка).
- **Где:** новое поле в замере стопы + логика toe-box в вердикте/оверлее.

## Осознанно отложено (решение зафиксировано, чтобы не передумывать молча)

### ⏸ Подъём / свод / объём стопы (instep)
Высота свода (low/normal/high), давит ли сверху.
- **Почему не сейчас:** это **третье измерение (высота)**, вынесено за рамки («2D, без подъёма»);
  по внутренней высоте колодки данных ещё меньше, чем по ширине → высокий риск выдумывания.
  Вернуться можно только если появится 3D или надёжный источник.

### ⏸ Петля обратной связи «реально подошло?»
Сбор «подошло/жало/болталось» после покупки → калибровка констант (`TOE_ROOM`, пороги зон,
сдвиги репутации) на реальных данных. В долгую превращает приближения в данные. Отложено.

### ⏸ Бренд-уровневые тенденции колодок
«Бренд N в среднем узит». Данные мягкие, и per-model `widthReputation` это уже покрывает —
отдельная ценность низкая.
