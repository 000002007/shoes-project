Веб-приложение-**примерка** кроссовок. Этот файл — **единый трекер прогресса**:
продукт, что сделано, что делать, решения и статусы. Обновляем по ходу.

> Статусы фаз: `⬜ не начато` · `🟡 в работе` · `✅ готово` · `⏸ потом`.

---

## Что это за продукт

**Примерка, а не подбор.** Пользователь выбирает конкретную модель и видит, как она
**сидит на его стопе** — через **рентген-оверлей**: силуэт стопы просвечивает под
контуром внутреннего пространства кроссовка, видно где тесно, где свободно.

**Поток на экране:**
1. **Рисунок (рентген-оверлей посадки)** — стопа vs внутренний контур модели; зоны
   «тесно / впору / свободно».
2. **Фидбэк** — словами: подходит / тесно / свободно + почему + уровень уверенности.

Это НЕ подбор и НЕ рекомендации («предложи мне модель» — потом). Реалистичное фото
«как смотрится на ноге» — отдельная необязательная опция на потом.

## Честное обещание

Примерка-ориентир: видишь приблизительную посадку модели на своей стопе + честную
оценку. Точного замера комфорта не обещаем — данные колодок производителей закрыты.

**Дисклеймеры (обязательны):**
- Контур кроссовка **приблизительный** (из размера + репутации ширины, не из колодок).
- Оверлей **2D** (длина × ширина). По литературе главные факторы посадки — **обхват (ball
  girth), объём и подъём** — оверлей НЕ видит: две стопы одной длины и ширины могут сидеть
  по-разному. Наша зона ширины — плоское **приближение (прокси)** к обхвату, который мы не меряем.
- Без ориентира (A4 на фото **или** введённого размера) оценка **недостоверна** —
  говорим прямо.

| Тема | Решение |
| --- | --- |
| Суть | Примерка: рентген-оверлей посадки → фидбэк. **Не подбор.** |
| Порядок на экране | рисунок (оверлей) → фидбэк |
| Рисунок | Рентген-оверлей: силуэт стопы vs внутренний контур модели; SVG/canvas, **без генеративного AI**, без оплаты за картинки |
| Откуда контур модели | приблизительно: длина из размера + ширина из репутации (узкая/стандартная/широкая) |
| Стек | Vite + React + Node-сервер (node:http) |
| LLM сейчас | заглушка (mock); потом Claude API + веб-поиск для атрибутов |
| Ключ API | на сервере, не во фронте |
| Фидбэк | строгие детерминированные правила; LLM только текст объяснения |
| Ввод стопы | фото с A4 **или** ручной ввод (длина/ширина/размер) с подписями против путаницы |
| Ориентир | есть A4/размер → оверлей+фидбэк; нет → предупреждение о недостоверности |
| Реалистичное фото «как выглядит» | необязательная опция, потом (генеративно, пометка «примерный вид») |
| База/каталог рекомендаций | потом |

## Снижение галлюцинаций (приоритет)

| Где | Метод |
| --- | --- |
| Атрибуты модели | структурированный вывод; запрет выдумывать числа → `unknown` + низкая уверенность; (потом) веб-поиск с источниками; подтверждение пользователем |

| Замер стопы | привязка к ориентиру (A4/размер); без него честно «недостоверно» |

### ⬜ Шаг 4 — Фидбэк (словами)
Цель: текстовый вердикт, **согласованный с картинкой**.
**Строится ПОВЕРХ шага 3:** вердикт потребляет уже готовый `FitResult` из `fitGeometry`
(те же зоны длины/ширины и `confidence`), а НЕ пересчитывает геометрию — так рисунок и
слова не разойдутся.

**Решения из исследования (заложены, см. [research/foot-and-shoe-notes.md](research/foot-and-shoe-notes.md)):**
- **Ширина — это прокси, не вердикт по обхвату.** Реальную тесноту решает обхват (ball girth),
  которого у нас нет. Поэтому `tight` по ширине → мягко («может быть тесновато по ширине»),
  gated по `confidence`; **запрещён безусловный «не подойдёт»**. Зелёное «впору» по ширине —
  «по плоской ширине запас есть, обхват не учтён».
- **Два ОРТОГОНАЛЬНЫХ сигнала материала** (не смешивать): `instantGive` из `attrs.stretch`
  (мгновенная податливость) и `breaksIn` из **семейства материала** (кожа/замша = да; трикотаж/
  синтетика/unknown = нет). Разнашивание — только кожа; синтетике честный контр-сигнал «жёсткий,
  теснота останется»; трикотаж «тянется, но не разнашивается навсегда»; unknown → молчим.
- **Смягчение — только ТЕКСТ по ширине.** Не мутирует зоны/запасы, не флипает `tight→fit`,
  длину не трогает. В данных `widthZone` остаётся `tight`, смягчается только формулировка.
- **Оценочная ширина (`widthEstimated`) / низкая `confidence`** → смягчение глушим, зовём
  «укажи ширину — оценим точнее». При конфликте/пропусках — смягчать МЕНЬШЕ.
- **Подошва/межподошва (амортизация, сцепление) — ВНЕ fit-вердикта** (это про ход, не про длину/ширину).

- [ ] **4.0 `src/fit/material.ts`.** Чистая функция `materialFamily(raw) → leather|knit|synthetic|unknown`
  по токенам (кожа/замша/leather/suede → leather; вязан/knit/flyknit/трикотаж → knit; mesh/сетка/
  текстиль/синтет/tpu → synthetic; иначе unknown). Только `leather` даёт `breaksIn`. _Тесты:_ строки фикстур.
- [ ] **4.1 Правила вердикта.** Чистая функция: из `FitResult` (зоны) + `instantGive`(stretch) +
  `breaksIn`(материал) → вердикт подходит/тесно/свободно, **по правилам выше**. Смягчённую причину
  пишем нотой в `FitResult.notes` (аудируемо). _Готово:_ функция + тесты (вкл. ловушки фикстур:
  кожа=rigid-но-breaksIn; Primeknit=stretchy-но-без-разнашивания).
- [ ] **4.2 Уровень уверенности.** Переиспользуем/расширяем `FitResult.confidence`. _Готово:_ функция + тесты.
- [ ] **4.3 Текст объяснения.** Из вердикта — 1–2 фразы причины (шаблоны/правила на MVP;
  LLM-формулировка потом). _Готово:_ функция возвращает понятный текст.
- [ ] **4.4 UI-блок фидбэка** под оверлеем: вердикт + бейдж уверенности + причина + дисклеймеры;
  нет ориентира → «недостоверно». _Готово:_ блок рисуется.
- [ ] **4.5 Сборка потока** на одном экране (модель → замер → оверлей → фидбэк), общее состояние.
- **Готово (шаг целиком), когда:** на (стопа + модель) выдаётся стабильный вердикт из 3 категорий
  + причина + уверенность, согласованный с оверлеем, с честной прокси-оговоркой по ширине.
