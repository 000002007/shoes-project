# shoes-project — Роадмап и трекер workflow

Веб-приложение для подбора кроссовок по удобству. Этот файл — **единый трекер
прогресса**: продукт, фазы, зафиксированные решения и статусы. Обновляем по ходу.

> **Как пользоваться:** у каждой фазы есть статус.
> `⬜ не начато` · `🟡 в работе` · `✅ готово` · `⏸ отложено (потом)`.

---

## Продукт: две фичи

- **Фича 1 (ядро, делаем первой):** примерный **фидбэк** — подойдут ли эти
  кроссовки (подойдёт / тесно / свободно). Правдивая часть: данные + правила.
- **Фича 2 (после):** **картинка** человека в этой обуви — наглядный слой.
  Косметика: всегда с пометкой «примерный вид, не отражает посадку».

Правда о посадке живёт в Фиче 1. Фича 2 — визуальный «вау», но не источник истины.

## Честное обещание

Подсказка о посадке на основе замеров стопы и репутации модели — **ориентир, а
не примерка**. Точного замера комфорта не обещаем (данные колодок закрыты).
Сильная сторона — сравнение с обувью, которую пользователь уже удобно носит.

**Обязательные дисклеймеры:**
- Без ориентира (A4 на фото **или** введённого размера) оценка **недостоверна** —
  говорим об этом прямо, крупно.
- Картинка в Фиче 2 — **примерный вид**, не показывает реальную посадку.

## Зафиксированные решения

| Тема | Решение |
| --- | --- |
| Стек | Vite + React + Node-сервер (как в wc26-predictor) |
| LLM сейчас | Заглушка (mock), без затрат |
| LLM потом | Claude API + веб-поиск (вариант b), переключение через env |
| Ключ API | На сервере, не во фронте |
| Вердикт | Строгие детерминированные правила; LLM только формулирует объяснение |
| Ввод стопы | Фото с A4 **или** ручной ввод (длина / ширина / размер) с подписями на сайте |
| Ориентир | Фидбэк только при A4 или размере; иначе — предупреждение о недостоверности |
| Антигаллюцинации | Уверенность + подтверждение пользователем + запрет выдумывать числа |
| Картинка (Фича 2) | Косметика, после Фичи 1, с пометкой «примерный вид» |
| База данных | Отложена: сначала кеш, потом каталог для рекомендаций |

## Снижение галлюцинаций (приоритет)

| Где | Метод |
| --- | --- |
| Атрибуты модели | структурированный вывод; запрет выдумывать числа → `unknown` + низкая уверенность; веб-поиск с показом источников; подтверждение пользователем; растущий кеш проверенных моделей как источник правды |
| Вердикт | детерминированные правила, без LLM в решении → ноль галлюцинаций по построению |
| Замер стопы | привязка к ориентиру (A4/размер); без ориентира честно «недостоверно», а не выдуманная цифра |
| Картинка (Фича 2) | признаём генеративную природу → не источник правды, пометка «примерный вид» |

## Фазы

### ⬜ Шаг 0 — Что обещаем пользователю
Честное обещание + дисклеймеры (без ориентира — недостоверно; картинка — примерный вид).
- **Готово, когда:** формулировка обещания и дисклеймеры зафиксированы в UI.

### ✅ Шаг 1 — Модель → атрибуты (LLM-скелет, без базы) — часть Фичи 1
**ГОТОВО** — реализовано и влито в `main` (FF). Тесты: сервер 16/16, фронт 8/8, сборка чистая; живой прогон сервера ОК.
Ввёл название модели → атрибуты + уверенность → правка/подтверждение.
Спека: [step1-model-attributes-design](superpowers/specs/2026-06-15-step1-model-attributes-design.md).
План: [step1-model-attributes](superpowers/plans/2026-06-15-step1-model-attributes.md).
- **Входит в MVP.** LLM — заглушка (mock), потом Claude API + веб-поиск.
- **Готово:** по названию модели система отдаёт нормализованный объект атрибутов
  с уверенностью, его можно подтвердить/поправить; ошибка → ручной ввод.
- **Заметки на шаг подключения Anthropic:** `getProvider()` сейчас вызывается при
  загрузке модуля (для теста переключения провайдера сделать ленивым); в провайдер
  уходит «сырое» имя модели (для веб-поиска это правильнее, чем нормализованное).

### ⬜ Шаг 2 — Ввод стопы: фото с A4 или ручной ввод — часть Фичи 1
- **Ручной ввод (обязательная опция):** длина, ширина, размер — отдельными полями.
- **Подписи на сайте против путаницы:** прямо пишем у каждого поля, например:
  «Длина стопы — от пятки до самого длинного пальца, в мм. Это НЕ длина стельки и
  НЕ размер обуви»; «Ширина — самое широкое место стопы».
- **Фото с A4** — референс масштаба (для замера и Фичи 2).
- **Логика ориентира:** есть A4 или размер → можно давать фидбэк; нет → крупное
  предупреждение о недостоверности, фидбэк помечается как ненадёжный или не выдаётся.
- **Входит в MVP.**
- **Готово, когда:** замер хранится нормализованно (мм + категория ширины),
  переводится в размеры систем; логика «ориентир или предупреждение» работает.

### ⬜ Шаг 3 — Фидбэк о посадке (вердикт на правилах) — Фича 1
Подойдёт / тесно / свободно + уверенность + причина + дисклеймер. Только при ориентире.
- **Входит в MVP. Это и есть Фича 1.**
- **Готово, когда:** на (стопа + модель) выдаётся стабильный вердикт из 3 категорий
  + уверенность + причина, с учётом наличия/отсутствия ориентира.

### ⬜ Шаг 3b — Честная визуализация посадки (достижимый визуал)
Схема: контур стопы vs внутренние размеры модели, цветовые зоны «где тесно».
Не генерация — почти не галлюцинирует.
- **Желательно в MVP** (усиливает Фичу 1 наглядно, дёшево).
- **Готово, когда:** на экране видно соотношение стопы и модели с зонами тесноты.

### ⏸ Шаг 4 — Картинка «обувь на ноге» — Фича 2 (после Фичи 1)
- **4a — генеративное совмещение** (фото ноги + референс кроссовка → картинка):
  прототип одним API, но недостоверно и платно. Опция с пометкой «косметика».
- **4b — AR/3D try-on (Wanna-style):** неподъёмно соло. Не делаем.
- Технику выберем на этапе Фичи 2.

### ⏸ Шаг 5 — База данных
Сначала кеш проверенных моделей (копится из подтверждений шага 1), затем каталог
для **рекомендаций** брендовых кроссовок.

## Граница MVP

**MVP = Фича 1:** цепочка `модель → атрибуты → стопа (A4/ручной) → фидбэк` (шаги
1–3) + честная визуализация (3b). Привязка к ориентиру и дисклеймеры — часть MVP.

**После MVP:** Фича 2 — картинка на ноге (шаг 4). **Потом:** база/каталог (5),
аккаунты, форма стопы (подъём, косточки), монетизация.

## Стек

Vite + React (фронт) + Node-сервер (прячет ключ). LLM через интерфейс провайдера:
заглушка → Claude API + веб-поиск. Хранилища на MVP нет. Вердикт — обычный код,
LLM только текст объяснения.
