**Дата:** 2026-06-15

## Цель и критерий готовности

Получить и честно сохранить размеры стопы пользователя в виде объекта
`FootMeasurement`, который позже питает **рентген-оверлей (шаг 3)** и **фидбэк
(шаг 4)**. Плюс — решить, есть ли надёжный ориентир, и честно предупредить, если нет.

**Готово, когда:** замер хранится нормализованно (длина в мм + категория ширины),
конвертируется между системами размеров, и работает логика «ориентир есть → можно
дальше / нет → предупреждение о недостоверности».

## Рамки

- **Только фронт** (Vite + React + TS, тесты Vitest). Сервер не трогаем — шаг чисто
  клиентский (состояние + чистая логика).
- **A4-фото с замером по фото — вне рамок этого шага** (подшаг 2.6 отложен): он
  сложнее (CV/калибровка) и для оверлея не нужен — хватает ручного ввода. Вернёмся позже.
- Не делаем: оверлей (шаг 3), фидбэк (шаг 4), сохранение в БД.

## Ключевые решения

- **Внутренняя единица — миллиметры** (Mondopoint = длина стопы в мм).
- **Одна стопа = бо́льшая.** Просим мерить большую ногу (подпись), храним один замер.
- **Ввод длины — в см** (как меряют линейкой, напр. 26.5), храним мм. Ширина — в см
  (опционально) **или** категория (узкая/стандартная/широкая). Размер — значение +
  система (EU / US муж. / UK), опционально.
- **Ориентир (`hasReference`)**: есть длина **или** размер. Только ширина — НЕ ориентир.
  Нет ориентира → крупное «оценка недостоверна». Нет ширины → пометка «по ширине грубо».
- **Конвертация размеров — приблизительная**, помечена как ориентировочная (точных
  таблиц у брендов нет; используем стандартную связь через парижский штих).

## Тип данных

```
WidthCategory = 'narrow' | 'standard' | 'wide'
SizeSystem    = 'EU' | 'US_M' | 'UK'
FootMeasurement = {
  lengthMm?: number          // длина стопы, мм (основной сигнал)
  widthMm?: number           // ширина, мм (если измерил)
  widthCategory?: WidthCategory  // если не мерил, но знает/выбирает
  sizeValue?: number         // размер, если знает
  sizeSystem?: SizeSystem
  source: 'manual' | 'a4'    // на этом шаге всегда 'manual'
}
```

## Конвертация размеров (приблизительная)

Через парижский штих (2/3 см), с типичным припуском колодки ~15 мм:
- `mm → EU`: `EU = round((lengthMm/10 + 1.5) * 1.5)` (напр. 265 мм → 42).
- `EU → mm`: `lengthMm = round((EU/1.5 - 1.5) * 10)`.
- `US_M ≈ EU − 33`, `UK ≈ US_M − 1` (грубое смещение для взрослых).
Все функции помечены «приблизительно» в комментариях и в UI.

## Поток данных

Пользователь вводит длину/ширину/размер → формируется `FootMeasurement` (длина в мм,
категория ширины выводится из мм при наличии, иначе берётся выбранная) → `hasReference`
решает, достоверно ли → предупреждения показываются явно. Объект остаётся в состоянии
приложения (питание шагов 3–4 — позже).

## Обработка краёв

- Пустые/нечисловые поля игнорируются (не ломают объект).
- Неадекватные значения (длина < 150 мм или > 400 мм) → мягкое предупреждение, не краш.
- Если задан и размер, и длина — длина приоритетна как сигнал; размер не перетирает.

## Тестирование (TDD)

- `sizing`: пары mm↔EU на контрольных точках; US/UK смещения; обратимость в пределах округления.
- `reference`: длина→ориентир есть; размер→есть; только ширина→нет; пустой→нет; набор предупреждений.
- `FootInput`: ввод длины обновляет замер; без ориентира виден баннер; подписи присутствуют.
