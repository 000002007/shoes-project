# Шаг 3 — Рентген-оверлей посадки (дизайн)

**Дата:** 2026-06-16
**Статус:** согласовано (владелец делегировал реализацию), ожидает плана реализации
**Трекер:** [ROADMAP](../../ROADMAP.md)

## Цель и критерий готовности

Наглядно показать, **как модель сидит на стопе**: полупрозрачный силуэт стопы под
внутренним контуром кроссовка **в одном масштабе**, с цветными зонами тесноты по длине
и ширине. Это **рисунок и геометрия** — текстовый вердикт и смягчение по растяжимости
верха это **шаг 4**.

**Готово, когда:** по (подтверждённая модель + замер стопы с ориентиром) на экране виден
силуэт стопы относительно внутреннего контура модели рекомендуемого размера, с подсветкой
тесных зон, честными пометками «приблизительно, 2D» и понятными пустыми состояниями.

## Зафиксированные решения (брейншторм 2026-06-16)

| Тема | Решение |
| --- | --- |
| Размер | **Авто: рекомендуемый размер** из длины стопы (`sizing.ts`) + поправка `sizeReputation` |
| Числа | **Адаптивная точность**: мало данных → только зоны + предупреждение; больше данных (ширина/A4) → можно «≈ диапазон»; всегда «приблизительно» |
| Форма | **Схематично**: сглаженный силуэт стопы + простой скруглённый контур обуви, один масштаб мм→px |
| Stretch | **НЕ в оверлее** — чистая геометрия; смягчение по `stretch` это слова шага 4 |
| Контур | **Подход A «репутация-как-сдвиг»**: номинал из размера + сдвиги по репутации в мм |

## Ограничения (зафиксированы)

- Вся геометрия — **чистые функции** (как шаги 1–2), рендер отдельно. **Ноль AI/генерации.**
- Колодки производителей закрыты → всё **приближение**, помечено. **Без ориентира не рисуем.**
- Реюз: `src/foot/sizing.ts` (`mmToEu`/`euToMm`/`mmToSize`/`sizeToMm`) и
  `src/foot/reference.ts` (`hasReference`). Состояние уже поднято в `App` (шаг 2/аудит).
- Все константы (припуск носка, сдвиги репутации, пороги зон, ratio ширины) — **в одном
  месте**, консервативные, с тестами; в UI помечены приблизительными.

## Архитектура и модули

**Чистые функции (`src/fit/`)**
- `contour.ts`
  - `recommendedSizeEu(footLenMm, sizeReputation) → number` — рекомендуемый размер EU.
  - `internalContour(footLenMm, attrs) → { internalLenMm, internalWidthMm }` — внутренний
    контур выбранного размера (подход A).
  - `constants` — все припуски/сдвиги/ratio в одном объекте с комментариями «приблизительно».
- `geometry.ts`
  - `fitGeometry(foot, attrs) → FitResult` — единая точка: длина/ширина стопы и контура,
    запасы, зоны, уверенность, честные пометки. Обрабатывает неполные данные.

**Компонент (`src/components/`)**
- `FitOverlay.tsx` — SVG-рендер `FitResult`: два силуэта в масштабе, цвет зон, легенда,
  пометки, пустые состояния.

**Встройка**
- `App.tsx`: когда есть подтверждённые `attributes` **и** `foot` с ориентиром → рисуем
  `<FitOverlay attrs={attributes} foot={foot} />` на месте нынешней строки «Готово к шагу 3».

## Типы

```ts
type Zone = 'tight' | 'fit' | 'loose' | 'unknown'

interface FitResult {
  drawable: boolean            // false → рисуем заглушку, не оверлей
  footLenMm?: number
  footWidthMm?: number         // измерена/оценена; undefined если совсем нет
  internalLenMm?: number
  internalWidthMm?: number
  recommendedSizeEu?: number
  lengthMargin?: number        // internalLen − footLen
  widthMargin?: number         // internalWidth − footWidth
  lengthZone: Zone
  widthZone: Zone
  widthEstimated: boolean      // ширина не измерена, а оценена → грубее
  confidence: 'high' | 'medium' | 'low'
  notes: string[]              // честные пометки/предупреждения
}
```

## Модель контура (подход A «репутация-как-сдвиг»)

1. **Длина стопы** `footLenMm`: из `lengthMm`; иначе из `sizeValue` через `sizeToMm`
   (грубее → пометка «длина из размера, менее точно»); иначе нет ориентира → `drawable=false`.
2. **Рекомендуемый размер** `recEu = round(mmToEu(footLenMm)) + repAdj`, где `repAdj`:
   `runs_small → +1`, `runs_large → −1`, `true_to_size`/`unknown → 0`.
   **Репутация размера учитывается ТОЛЬКО здесь** (чтобы не считать дважды); остаточный
   сигнал по длине = зазор округления стопы до доступного размера.
3. **Внутренняя длина** `internalLenMm = euToMm(recEu) + TOE_ROOM`.
4. **Внутренняя ширина** `internalWidthMm = WIDTH_RATIO * internalLenMm + widthRepShift`,
   где `widthRepShift`: `narrow → −W`, `wide → +W`, `standard`/`unknown → 0`.
5. **Ширина стопы** `footWidthMm`: из `widthMm`; иначе из `widthCategory`
   (narrow/standard/wide → ratio-оценка); иначе нейтральный `FOOT_WIDTH_RATIO * footLenMm`
   с флагом `widthEstimated=true` и пометкой «ширина не задана, грубо».
6. **Запасы:** `lengthMargin = internalLenMm − footLenMm`, `widthMargin = internalWidthMm − footWidthMm`.

**Стартовые константы (калибруются тестами в плане, помечены приблизительными):**
`TOE_ROOM ≈ 10 мм`; `WIDTH_RATIO ≈ 0.40`; `FOOT_WIDTH_RATIO ≈ 0.38`; `widthRepShift W ≈ 6 мм`;
категории ширины стопы → доли длины (narrow ≈ 0.36, standard ≈ 0.38, wide ≈ 0.41).

## Зоны и пороги

`Zone` из запаса по порогам (стартовые, калибруются):
- **Длина:** `margin < 4 → tight`; `4..14 → fit`; `> 14 → loose`.
- **Ширина:** `margin < 2 → tight`; `2..10 → fit`; `> 10 → loose`.

Длина и ширина — **раздельно** (две зоны). Шаг 3 их только показывает; сводный вердикт — шаг 4.

## Адаптивная точность и уверенность

- **Уверенность `confidence`** падает (high→medium→low) при: ширина не измерена
  (`widthEstimated`); длина выведена из размера, а не задана; `sizeReputation`/
  `widthReputation = unknown`; низкая `attributes.confidence`.
- **Больше данных → точнее:** при измеренной ширине / длине-в-мм допуск уже, в подписи зоны
  можно показать «≈ диапазон» запаса (напр. «запас по длине ≈ 5–10 мм»). Иначе — **только
  зона** + подсказка «укажи ширину или замерь по A4 — будет точнее».
- **Неизвестная репутация** (`unknown`) → берём нейтрально (`repAdj=0`, `shift=0`) + пометка
  «репутация неизвестна, точность ниже».

## Рендер (SVG)

- Масштаб **мм→px** от наибольшей длины (внутренняя длина) в фиксированный `viewBox`; обе
  фигуры — в нём, в одном масштабе.
- Стопа — полупрозрачный сглаженный силуэт (по длине×ширине); обувь — скруглённый контур
  поверх. Выровнены по пятке.
- Красным — где силуэт стопы **выходит за** контур (tight-зоны). Нейтрально — впору/свободно.
- Снизу: **легенда** (тесно/впору/свободно) + **пометки** из `notes`.
- Никакого AI/генерации; чистый SVG из чисел `FitResult`.

## Обработка ошибок и пустые состояния

- **Нет ориентира** (`hasReference=false`): `drawable=false` → заглушка «Укажи длину стопы
  или размер — тогда покажем посадку», без рисунка.
- **Нет подтверждённой модели**: оверлей не показываем (App не рендерит `FitOverlay`).
- **Ширина не задана**: рисуем со средней оценкой, зона ширины помечена «грубо/оценка»,
  `confidence` снижена.
- **Репутация `unknown`**: нейтрально + пометка.
- Всегда внизу: «Приблизительно, 2D — без подъёма и косточек; колодки производителей закрыты».

## Тестирование (TDD)

- **`contour`**: `recommendedSizeEu` (runs_small двигает вверх, runs_large вниз,
  true_to_size/unknown без сдвига); `internalContour` (narrow → у́же, wide → шире; длина
  растёт с размером); инвариант «при true_to_size и стопе ровно под размер — запас ≈ TOE_ROOM».
- **`geometry`**: кейсы `tight`/`fit`/`loose` по длине и ширине; неполные данные (нет ширины
  → `widthZone` помечена/`unknown`, `widthEstimated=true`; нет длины, есть размер → через
  `sizeToMm` + пометка; нет ориентира → `drawable=false`); `unknown`-репутация → нейтраль +
  пометка; `confidence` снижается по правилам.
- **`FitOverlay`**: рисует обе фигуры; tight-зона красная; легенда и пометки на месте;
  пустое состояние без ориентира (заглушка, нет `<svg>` посадки).
- Всё тестируется без сети и без ключа API.
